# 457DEEP 공기업 면접 강의 시리즈 제작 스킬

역할: input 폴더에 있는 주제와 특별히 주문하는 내용을 읽고, 이를 반영해서 output 폴더에 docx 형태로 출력한다

## 1. 시리즈 개요

- 브랜드: 457deep 취업 컨설팅
- 콘텐츠: 공기업 면접 강의 시리즈 (총 39편, 9개 군)
- 타깃: 20~30대 공기업 취업 준비생 (NCS 기반 구조화 면접 준비자)
- 출력 형식: docx (python-docx 또는 docx-js 사용)

## 2. 시리즈 구성표 (39편)

| 군 | 편 | 주제 |
|---|---|---|
| 1군. 오리엔테이션 | 1~2편 | 1분 자기소개, 공기업 면접 11가지 특수성 |
| 2군. 기관 이해/지원동기 | 3~10편 | 왜 공기업, 왜 이 분야, 왜 이 기관(A형/B형), 기관 정의, 관심 사업, 위기와 기회, 입사 후 목표 |
| 3군. 직무 이해 | 11~15편 | 직무 선택 이유, 직무 역량, 직무 관련 경험, 직무 트렌드 |
| 4군. 인성/자기 이해 | 16~18편 | 성격 장단점, 갈등 경험, 실패 경험 |
| 5군. NCS 경험 질문 | 19~26편 | 의사소통, 문제해결, 자원관리, 조직이해, 대인관계, 정보능력, 기술능력, 수리능력 |
| 6군. 공직 가치/직업윤리 | 27~30편 | 청렴, 공익, 윤리 딜레마, 공정성 |
| 7군. 민원 응대 | 31~33편 | 악성 민원, 규정 vs 편의, 특수 민원 |
| 8군. 상황 면접 | 34~37편 | 출제 유형 6가지, 대응 공식 9가지, 프레임워크, 표현 모음 |
| 9군. 마무리 | 38~39편 | 마지막 할 말, 면접 후 태도 |

## 3. 말투/어투 규칙

### 3-1. 기본 어조
- 강의체 사용: "~합니다", "여러분", "~하시기 바랍니다"
- 직설적이고 구체적인 톤 (애매하게 돌려 말하지 않음)
- 공감체 사용: 취준생의 막막함, 긴장감을 인정하고 공감하는 톤
- 가볍게 유머를 섞되 과하지 않게

### 3-2. 절대 금지 표현
- 사기업 용어: "매출", "KPI", "경쟁력", "공격적", "시장 점유율", "영업", "고객 유치"
- 답변 예시 마무리: "적응하겠다", "배우겠다", "성장하겠다" 금지
- 서식: 이탤릭(기울임) 절대 금지
- 부호: em dash 절대 금지, 하이픈(-) 사용
- 표현: "축" 단어로 카테고리 표현 금지 (예: "3가지 축" 금지)
- 종결형 어미: "~봅니다" 금지

### 3-3. 선호 표현/키워드
- 공기업: 공익, 공정성, 청렴, 절차, 원칙, 투명성, 공공성, 사회적 가치
- 대상: 국민, 가입자, 수혜자, 이해관계자
- 기관: 경영 목표, 중장기 계획, 사업 방향, 기관장 취임사, 경영 공시
- 답변 마무리: "기여하겠다", "보태겠다", "수행하겠다" (기여/실행 톤)

### 3-4. 사기업 용어 -> 공기업 대체어

| 금지 | 대체 |
|---|---|
| 매출 | 사업 성과, 수행 실적 |
| KPI | 핵심 업무 지표, 성과 기준 |
| 경쟁력 | 전문성, 차별적 역량 |
| 공격적 (전략/마케팅) | 적극적, 선제적 |
| 시장 점유율 | 서비스 커버리지, 수혜 범위 |
| 영업 | 사업 수행, 업무 추진 |
| 고객 유치 | 가입자 확대, 수혜자 확대 |

## 4. 내용 구성 규칙

### 4-0. MECE 관점 풀이 원칙 (모든 편에 공통 적용)
- 질문을 분석할 때 반드시 MECE(Mutually Exclusive, Collectively Exhaustive) 관점에서 분류하여 풀이한다
- 분류 기준을 하나로 통일한다 (상황/역량/관계 등이 뒤섞이면 안 됨)
- 각 분류가 겹치지 않고, 전체를 빠짐없이 포괄하는지 검증한다
- 분류별로 프레임워크를 제시하고, 분류별로 예시를 제공한다
- 예시는 반드시 기관과 직무를 다양화한다 (4-5 기관/직무 풀 참조)

### 4-1. 답변 예시 작성 규칙
- 행정직/기술직/금융직 매칭: 예시마다 어떤 직군인지 명시
- 형식: "예시 X) [기관명] - [직무]"
- 수치 빈칸 처리: 구체적 수치는 "ㅇㅇ%", "ㅇㅇ명"으로 표기
- 치명적 단점 / 위장된 강점: 답변 예시에서 절대 사용 금지
- 두괄식 필수: 첫 문장에서 답변의 핵심을 먼저 던지기
- 문장 패턴 다변화: 예시마다 첫 문장 출발점, 전개 흐름, 종결 어미를 다르게
- 같은 패턴 반복 금지: "~에 관심이 있습니다"로 시작하는 예시가 연속되면 안 됨

### 4-2. 지원동기 답변 규칙
- 두괄식 필수
- 마무리: 기여/실행 톤 ("기여하겠다", "보태겠다", "수행하겠다")
- "적응하겠다", "배우겠다", "성장하겠다" 마무리 금지
- 연결고리 필수: 기관 사업/과제와 본인 경험/역량 연결
- 지원동기 5가지 유형: 기관 목표 기여형 / 직무 전문성 발휘형 / 유사 기관 차별화형 / 기관 변화-신사업 기여형 / 직업관-가치관 일치형

### 4-3. 기관 정의 답변 규칙
- 두괄식 필수: 첫 문장에서 기관 정의를 먼저 던지기
- 마지막 문장: "~라고 정의합니다"가 아니라 구체적 업무 기여로 마무리
- 3가지 관점: 수혜자 관점 / 기능 본질 관점 / 1차-2차 고객 프레임 관점

### 4-4. 상황면접 답변 규칙

출제 유형 6가지:
1. 관계 갈등/소통 상황
2. 변화 저항/설득 상황
3. 윤리/원칙 충돌 상황
4. 민원/고객 응대 상황
5. 자원 부족/위기 대응 상황
6. 업무 실수/사후 수습 상황

대응 9가지 공식:
1. 빌런은 없다 - 상황이 사람을 그렇게 만든 것
2. 설득과 대안이 통하는 사람으로 설정
3. 긍정적 언어로 전달
4. 자기 성찰 먼저
5. 관계별 선 지키기 (상사/동료/후배/협력업체)
6. 조직 친화/조직 중심
7. 소통은 정확하게 (회사 언어)
8. 원칙은 5천만 국민과의 약속
9. 일관성 유지 or 상황에 맞는 대안 제시

절대 금지 답변 3가지:
1. "매뉴얼대로 하겠습니다" -> 구체적 HOW를 말해야 함
2. "상사님께 보고하겠습니다" / "다른 상사에게 조언" -> 당사자와 직접 해결
3. 사적 해결 ("술 한잔 먹자") -> 비즈니스 절차로 해결

통합 답변 프레임워크 (5단계):
1. [HOW] ㅇㅇ 하겠습니다. (구체적 방법)
2. [상황 인지] 해당 상황의 핵심은 ㅇㅇ에서 기인한 것이라고 생각합니다.
3. [가치관/해결 방향] 이를 해결하기 위해 중요한 것은 ㅇㅇ라고 생각합니다.
4. [리스크] 만약 ㅇㅇ 한다면 ㅇㅇ 문제가 발생할 수 있습니다.
5. [상대방 고려 + 대안] 그러나 ㅇㅇ도 고려해야 하기 때문에, ㅇㅇ 대안을 드리겠습니다.

### 4-6. 관심 사업/이슈 답변 규칙

관심 사업 선택 기준 4가지 (최소 2가지 충족):
1. 기관이 현재 가장 중요하게 생각하는 사업인가
2. 현재 사회 문제/이슈 해결에 중요한 사업이라고 본인이 판단하는가
3. 본인 경험과 자연스럽게 연결고리가 만들어지는 사업인가
4. 본인의 역량으로 바로 기여 가능한 사업인가

기준 2개 조합별 프레임워크를 각각 제시하고, 조합별 예시 2개씩 제공한다.

관심 이슈 관점 5가지 (MECE 분류):
1. 기관 내부에서 발생한 이슈 - 기관이 스스로 추진하는 변화, 신사업, 조직 개편
2. 정책/제도 변화로 인한 이슈 - 정부 정책, 법 개정, 제도 변화가 기관 업무에 영향
3. 사회/환경 변화로 인한 이슈 - 인구구조, 기술 변화, 기후위기 등 외부 환경
4. 기관 간 협업/경계 변화로 인한 이슈 - 역할 범위 변화, 기관 간 협업 구조
5. 기관 신뢰/국민 인식 관련 이슈 - 꼬리 질문 대응용으로만 준비 (직접 꺼내지 않음)

관점 1 > 관점 2 > 관점 4 > 관점 3 순으로 안전. 관점 3은 단독으로 쓰지 말고 관점 1이나 2의 배경 설명으로만 활용.

이슈 답변 프레임워크:
1. [관심 이슈] 두괄식
2. [기관 영향] 이 이슈가 기관 업무에 어떤 영향을 주는지 (본인의 해석)
3. [기관 과제] 따라서 기관이 어떻게 해야 한다고 생각하는지 (본인의 관점)
4. [본인 역할] 본인 직무에서 어떻게 연결되는지

[3]의 표현 다변화 필수:
- "ㅇㅇ 기관이 해야 할 일은 ㅇㅇ이라고 생각합니다"
- "ㅇㅇ 기관의 핵심 과제는 ㅇㅇ이라고 생각합니다"
- "ㅇㅇ 기관이 나아가야 할 방향은 ㅇㅇ이라고 생각합니다"
- "ㅇㅇ 기관이 지금 집중해야 할 과제는 ㅇㅇ이라고 생각합니다"
- "ㅇㅇ 기관이 이 구조에서 맡아야 할 역할은 ㅇㅇ이라고 생각합니다"

이슈를 찾는 방법 10가지:
1. 기관 홈페이지 - 기관장 취임사, 중장기 경영 목표, 최근 1년 보도자료
2. 기관 경영공시 - alio.go.kr의 경영공시 정보
3. 주무부처 보도자료 - 기관을 관할하는 정부 부처의 최근 정책 발표
4. 관련 법령 개정안 - 기관 업무에 직접 영향을 주는 법령의 최근 개정/입법 예고
5. 언론 보도 - 기관명으로 최근 3개월 뉴스 검색
6. 국정감사 지적 사항 - 매년 10월 국정감사에서 해당 기관에 대해 지적된 사안
7. 경영평가 결과 - 기획재정부 공공기관 경영평가에서 해당 기관의 항목별 점수
8. 기관 발간 보고서/백서 - 경영보고서, 지속가능경영보고서, 연차보고서
9. 국회 입법조사처/예산정책처 보고서 - 해당 기관 관련 정책 이슈 분석 보고서
10. 기관 노동조합 보도자료/성명서 - 조직 내부 쟁점 파악

## 5. 출력 포맷 규칙 (마크다운 -> docx 변환 시 기준)

### 5-1. 이모지
- 모든 큰제목/소제목 앞에 이모지 1개 필수

### 5-2. 제목 서식
- 마크다운 헤더(#, ##, ###, #### 등) 절대 사용 금지
- 큰제목/소제목은 이모지 + 텍스트로만 구분 (예: "📌 기준 3) 본인 경험과 연결고리")

### 5-3. 서식 금지
- 이탤릭 절대 금지
- em dash 절대 금지, 하이픈(-) 사용

### 5-4. 섹션 전환
- 섹션 사이: `&nbsp;` + `---` + `&nbsp;` (빈줄 + 구분선 + 빈줄)

### 5-5. 리스트/인용블록
- 리스트, 인용블록 앞뒤에 `&nbsp;` 필수
- 답변 예시: 인용블록(`>`) 처리
- 프레임워크: 인용블록(`>`) 처리
- 좋은 예/나쁜 예: 인용블록(`>`) 처리

### 5-6. 답변 예시 형식
- "예시 X) [기관명] - [직무]" 형식
- 행정직/기술직/금융직 등 직군 명시
- 수치는 "ㅇㅇ%", "ㅇㅇ명" 처리

## 6. docx 출력 규칙

### 6-1. 기본 설정
- 용지: A4
- 폰트: Arial 또는 맑은 고딕
- 본문 크기: 11pt
- 제목 크기: 16pt (H1), 14pt (H2), 12pt (H3)
- 줄간격: 1.5
- 여백: 상하좌우 2.54cm (1인치)

### 6-2. docx 구조
- 표지: "457DEEP 공기업 면접 강의 시리즈 - [편 제목]"
- 본문: 마크다운 포맷 규칙을 docx 서식으로 변환
- 답변 예시: 회색 배경 박스 또는 들여쓰기로 구분
- 표: 얇은 테두리, 헤더 행 배경색 처리

### 6-3. docx-js 사용 시 주의사항
- `npm install -g docx` 로 설치
- 유니코드 불릿 직접 입력 금지 -> `LevelFormat.BULLET` 사용
- `\n` 사용 금지 -> 별도 Paragraph 요소
- 테이블: `WidthType.DXA` 사용 (PERCENTAGE 금지)
- 테이블: `columnWidths` + 셀별 `width` 양쪽 다 설정
- 셀 배경: `ShadingType.CLEAR` 사용 (SOLID 금지)
- 생성 후 반드시 `python scripts/office/validate.py` 로 검증

## 7. 편별 제작 구조

### 7-1. 본편 구조
1. 시작 고정 문구 (5-7 참조)
2. 도입부: 이 편에서 다룰 내용 1~2문장 요약
3. 본문: 큰 주제별 이모지 + 제목 + 설명 + 예시
4. 답변 예시: 인용블록(>) 처리, 기관명/직군 명시, 수치 빈칸
5. 마무리: 이 편 핵심 요약 2~3줄 + 다음 편 예고
6. 끝 고정 문구 (5-8 참조)

### 7-2. 예시 작성 시 체크리스트
- [ ] 두괄식으로 시작했는가?

- [ ] 이탤릭, em dash가 사용되지 않았는가?
- [ ] 수치에 ㅇㅇ 빈칸 처리가 되어 있는가?
